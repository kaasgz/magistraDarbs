# Reproducibility Audit

Audit date: 2026-05-10.

Scope: the final thesis workflow that rebuilds the mixed real/synthetic
algorithm-selection dataset, selector results, thesis tables, figures, and
dashboard inputs. Development shortcuts are useful for smoke checks, but they
are not the source of the final thesis numbers.

## Current Verdict

The practical part is reproducible from the checked-in code, the configured
local XML inputs, and the documented command order. The workflow uses explicit
configuration files, fixed random seeds, isolated output folders, and run-summary
JSON files that capture settings, inputs, outputs, results, Python version, and
platform metadata.

The Python dependencies used by the project are pinned in `requirements.txt`.
For strict archival reproduction, a lock file or exported environment record can
also be kept together with the final run artifacts.

## Canonical Command Sequence

Run from the repository root after creating the virtual environment and
installing dependencies from `requirements.txt`.

```powershell
.\.venv\Scripts\python.exe -m src.experiments.generate_synthetic_dataset --n 180 --seeds 42,43,44 --output-root data\raw\synthetic\study
.\.venv\Scripts\python.exe -m src.experiments.run_real_pipeline_current --config configs\real_pipeline_current.yaml
.\.venv\Scripts\python.exe -m src.experiments.build_solver_compatibility_matrix
.\.venv\Scripts\python.exe -m src.experiments.run_synthetic_study --config configs\synthetic_study.yaml
.\.venv\Scripts\python.exe -m src.selection.build_selection_dataset_full
.\.venv\Scripts\python.exe -m src.selection.train_selector --full-dataset
.\.venv\Scripts\python.exe -m src.selection.evaluate_selector --full-dataset
.\.venv\Scripts\python.exe -m src.experiments.thesis_report
.\.venv\Scripts\python.exe -m src.thesis.generate_assets
```

This sequence rebuilds the artifacts consumed by the dashboard. Do not mix it
with `run_all_pipeline`, `run_thesis_pipeline`, or `src.web.app --bootstrap-demo`
when reproducing final thesis results; those commands are development helpers.

## Inputs

| Input | Location | Reproducibility contract |
| --- | --- | --- |
| Real RobinX / ITC2021 XML | `data/raw/real/` | Must be present before running the real pipeline. The project does not download these files. The current local thesis run expects 54 XML files. |
| Synthetic study XML | `data/raw/synthetic/study/` | Regenerated by the first canonical command with `--n 180 --seeds 42,43,44`. |
| Real pipeline config | `configs/real_pipeline_current.yaml` | Fixes input path, output paths, seed `42`, 60-second solver limit, repeated 3x3 validation, and unconfigured Timefold behavior. |
| Synthetic study config | `configs/synthetic_study.yaml` | Fixes seeds `42,43,44`, 60-second solver limit, solver portfolio, repeated 3x3 validation, and output folders. |
| Selector config | `configs/selector_config.yaml` | Fixes full-dataset paths, selector seed `42`, Random Forest model choice, and repeated 3x3 validation. |

## Randomness And Splits

| Stage | Randomness control |
| --- | --- |
| Synthetic XML generation | Dataset seeds `42,43,44`; generated metadata uses a fixed timestamp. |
| Real benchmark pipeline | `run.random_seed: 42` in `configs/real_pipeline_current.yaml`. |
| Synthetic study benchmarks | Benchmark seeds `42,43,44` in `configs/synthetic_study.yaml`. |
| Selector training/evaluation | `run.random_seed: 42` in `configs/selector_config.yaml`. |
| Validation | `repeated_stratified_kfold`, 3 folds, 3 repeats. |
| Solver time budget | 60 seconds per solver run unless a solver-specific override is configured. |

The random seed is passed through the pipeline and to solver/model components
where they support seeded behavior. Time-limited optimization can still show
small machine-dependent differences when a solver reaches its wall-clock budget.

## Audit Notes

- Real and synthetic outputs are isolated in separate folders before the mixed
  full-selection artifacts are built.
- Run-summary JSON files preserve configuration snapshots and path metadata for
  the major stages.
- `timefold` is part of the registered portfolio. With `executable_path: null`,
  its rows are explicitly recorded as `not_configured`; this is reproducible and
  should not be interpreted as a failed local solver run.
- The dashboard is read-only over generated artifacts. It does not rerun
  experiments when the page loads.
- The project should keep the final XML input set stable. For a stricter archive,
  store checksums for `data/raw/real/*.xml` and the final generated synthetic XML
  files.
- Dependency versions should be pinned before final hand-in if exact
  environment-level reproduction is required.
